# Fintech UI Patterns

Use this skill for financial product interfaces where user trust and interpretation accuracy matter.

## Sources

- VoltAgent awesome-design-md fintech references: https://github.com/VoltAgent/awesome-design-md
- Relevant inspirations: Stripe, Coinbase, Revolut, Wise, Kraken, Binance, Mastercard

## Principles

- Make balances, holdings, risk, and cash movement unambiguous.
- Pair every amount with currency, timeframe, and status when ambiguity is possible.
- Make sensitive values maskable and avoid casual exposure of account identifiers.
- Use clear confirmation steps for transfers, trades, payment changes, and destructive actions.
- Disclose fees, settlement timing, exchange rates, slippage, and risk before commitment.
- Use trustworthy visual language: calm surfaces, strong hierarchy, precise copy, and restrained celebration.
- Treat red/green financial color carefully; support colorblind interpretation with signs, labels, or icons.

## Review Checklist

- Primary financial values are readable at a glance.
- Gains, losses, pending states, and failed states are visually distinct and textually clear.
- The interface never hides fees or material timing.
- Risky actions have review, confirmation, and recovery paths.
- Security signals are present without creating clutter.
- Charts and tables remain legible in dark mode and on mobile.
