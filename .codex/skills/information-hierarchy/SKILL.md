# Information Hierarchy

Use this skill to decide what users should see first, second, and later.

## Sources

- VoltAgent awesome-design-md layout and typography sections: https://github.com/VoltAgent/awesome-design-md
- shadcn/ui Theming token roles: https://ui.shadcn.com/docs/theming

## Principles

- Organize by user intent before visual style.
- Use size, position, contrast, spacing, grouping, and motion in that order of restraint.
- Prefer fewer, clearer hierarchy levels over many competing weights.
- Put global context near the top: account, timeframe, mode, filters, or status.
- Group related values, labels, actions, and explanations so the user does not have to hunt.
- Keep secondary actions visible but visually quieter than primary work.

## Review Checklist

- The page has one dominant reading path.
- Each section has a clear job and a meaningful heading or label.
- Repeated cards or rows use stable structure and comparable alignment.
- Important warnings or data quality notes appear before decisions are made.
- Tables, charts, and lists expose enough context without forcing deep reading.
