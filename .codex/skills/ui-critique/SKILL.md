# UI Critique

Use this skill to review product UI with a practical, implementation-aware eye.

## Sources

- VoltAgent awesome-design-md guardrails and responsive sections: https://github.com/VoltAgent/awesome-design-md

## Critique Order

1. Goal fit: identify what the screen is trying to help the user do.
2. Hierarchy: check whether the eye lands on the right thing first.
3. Legibility: inspect typography, contrast, density, and data formatting.
4. Interaction: review focus, hover, keyboard, loading, empty, error, and disabled states.
5. Responsiveness: verify layout stability across mobile and desktop.
6. System fit: compare components, tokens, radius, spacing, and icon style.

## Principles

- Lead with concrete issues and likely user impact.
- Prefer small actionable fixes over vague aesthetic advice.
- Flag overdecorated UI, weak contrast, crowded dashboards, hidden filters, and inconsistent spacing.
- Treat accessibility as product quality, not a final pass.
- In fintech, prioritize trust, data clarity, and error prevention over visual novelty.

## Review Checklist

- The primary action and primary information are obvious.
- Text fits and remains readable at realistic viewport sizes.
- Components do not shift when state or content changes.
- Theme, spacing, radius, icons, and charts feel from the same system.
- The critique names what to change, where, and why.
